Day 5: Git & GitHub

What is Git?

Git is a version control system that tracks changes in your code.

What is GitHub?

GitHub is a website where you store your Git repositories online.

1. Git Basics

# Initialize a new repo
git init

# Check status
git status

# Add files to staging
git add index.html
git add .           # add all files

# Commit changes
git commit -m "Initial commit"

# See commit history
git log
git log --oneline

2. Working with GitHub

# Clone a repo from GitHub
git clone https://github.com/username/repo.git

# Add remote (connect local to GitHub)
git remote add origin https://github.com/username/repo.git

# Push to GitHub
git push -u origin main

# Pull latest changes
git pull origin main

3. Branching & Merging

# Create a new branch
git branch feature-login

# Switch to a branch
git checkout feature-login
# OR create and switch in one command
git checkout -b feature-login

# List branches
git branch

# Merge a branch into current branch
git checkout main
git merge feature-login

# Delete a branch
git branch -d feature-login

4. Merge Conflicts

When two people edit the same file, Git shows this:

<<<<<<< HEAD
console.log("My change");
=======
console.log("Their change");
>>>>>>> feature-branch

Fix it by keeping the right code, removing the markers, then:

git add file.js
git commit -m "Resolved merge conflict"

5. .gitignore

Files to ignore in your repo:

node_modules/
.env
*.log
.DS_Store

6. GitHub Pages

Deploy a static site for free:

  1. Push your code to a GitHub repo
  2. Go to repo Settings → Pages
  3. Select branch "main" and folder "/ (root)"
  4. Your site is live at: https://username.github.io/repo-name

7. Pull Requests & Forks

  • Fork: Copy someone else's repo to your account
  • Pull Request: Ask to merge your changes into their repo
  • Steps: Fork → Clone → Make changes → Push → Create PR
